What's in it?

The docs are organized around Claude's core APIs and unique features. Key differences from OpenAI's API are noted.

  • 📨
    Messages API Core endpoint for sending prompts to Claude. Unlike OpenAI, the system prompt is a top-level field — not inside messages[] — keeping behavioral instructions cleanly separated from conversation history.
  • 🧠
    Extended Thinking Pass "thinking": {"type": "enabled", "budget_tokens": N} and Claude produces a visible thinking block — its internal scratchpad — before the final response. Unlike OpenAI's o-series, the reasoning is fully visible to you for debugging.
  • 💾
    Prompt Caching Cache repeated content (system prompts, large documents) server-side. Cached tokens cost up to 90% less — one parameter change, immediate savings on every repeated request.
  • 🛠️
    Tool Use Same concept as OpenAI's function calling. The Claude-specific docs show the exact JSON format differences — useful to compare side-by-side to deeply understand the pattern.
  • ⚡
    Batch API Process requests asynchronously at 50% off. Supports up to 300,000 output tokens per batch — ideal for bulk document processing, classification, and data pipelines.

Extended Thinking: Claude's reasoning mode

Enable Extended Thinking by adding "thinking": {"type": "enabled", "budget_tokens": 5000} to your API request. Claude will then produce a thinking block (its internal scratchpad) before the final response. Unlike OpenAI's o-series where reasoning is hidden, Claude's thinking is visible to you — you can log it, display it to advanced users, or use it for debugging. On tasks like multi-step code generation, math, or architectural decisions, Extended Thinking can lift quality as much as upgrading from Sonnet to Opus would.

Tips for using this resource effectively

  • Start with the Quickstart page — it provides working Python and TypeScript examples that get you to a functional API call in minutes and clearly shows the structural differences from OpenAI's API.
  • Read the Models overview early and bookmark it. The 2026 lineup is Haiku 4.5 ($1/$5, Adaptive Thinking), Sonnet 4.6 ($3/$15, 1M context), and Opus 4.7 ($5/$25, 1M context, highest capability). For most features, Sonnet 4.6 hits the best cost-to-capability ratio.
  • Try Extended Thinking on any task where you're not getting good enough results from your current model. It's often cheaper to enable Extended Thinking on Sonnet 4.6 than to upgrade to Opus 4.7, and the quality improvement is comparable.
  • If your app sends the same system prompt or document on every request, the Prompt Caching feature can cut those tokens' cost by up to 90%. The setup is a single additional parameter — it's worth implementing from day one.
  • Compare the Tool Use documentation with OpenAI's function calling docs side by side. The concepts are near-identical, and seeing both implementations helps you understand the pattern more deeply than reading either one alone.

A practical introduction to the Anthropic API using Python — covers the Messages API structure, model selection, tool use basics, and how Claude compares to OpenAI's API from a developer's perspective.

Quick Start: Messages API + Extended Thinking (Python)

The system parameter is top-level (not in messages[]). The second example shows Extended Thinking — Claude's visible reasoning mode. Install with pip install anthropic.

import anthropic # Initialize the client — API key from console.anthropic.com client = anthropic.Anthropic(api_key="sk-ant-...") # --- Standard message (Haiku 4.5 — cheapest, $1/$5 per 1M) --- message = client.messages.create( model="claude-haiku-4-5-20251001", max_tokens=1024, system="You are a helpful coding assistant.", messages=[ {"role": "user", "content": "Explain what an API is in two sentences."} ] ) # Response text lives in message.content[0].text print(message.content[0].text) print(f"Tokens — in: {message.usage.input_tokens}, out: {message.usage.output_tokens}") # --- Extended Thinking (Sonnet 4.6 — reasoning visible to developer) --- thinking_msg = client.messages.create( model="claude-sonnet-4-6", # $3/$15 per 1M — 1M context window max_tokens=8000, thinking={"type": "enabled", "budget_tokens": 5000}, messages=[{ "role": "user", "content": "Design the database schema for a multi-tenant SaaS app." }] ) for block in thinking_msg.content: if block.type == "thinking": print(f"[Thinking] {block.thinking[:200]}...") # visible reasoning elif block.type == "text": print(block.text) # final answer

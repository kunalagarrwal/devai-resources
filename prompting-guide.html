Prompt Engineering Guide

← Back to all resources

URL: https://www.promptingguide.ai

What is this resource?

The Prompt Engineering Guide (promptingguide.ai) is a comprehensive, community-maintained reference for writing effective prompts for large language models. It covers everything from basic prompting principles to advanced techniques used in research and production applications. Unlike the official API documentation from OpenAI or Anthropic — which focuses on the technical mechanics of calling the API — this guide focuses specifically on how to communicate with the model effectively once you have it working. It's one of the most thorough and well-organized resources on prompt engineering available for free.

What's in it?

The guide is organized into a progression from introductory to advanced content. The introductory sections cover what prompting is, how LLMs process input, and the basic elements of a well-structured prompt (instruction, context, input data, output format). From there, it covers specific prompting techniques including zero-shot prompting (giving the model a task with no examples), few-shot prompting (giving examples of the desired input-output format), and chain-of-thought prompting (asking the model to show its reasoning step by step before giving a final answer). The advanced sections cover techniques like self-consistency, tree-of-thought reasoning, and ReAct (reasoning and acting), which are used in more complex agent-based systems.

The site also has sections dedicated to specific applications — prompting for code generation, data analysis, and reasoning tasks — and a growing collection of model-specific guides that cover the quirks and best practices for individual models like GPT-4 and Claude.

How is it relevant to your purpose?

Getting the API working is only half the challenge when integrating AI into an application. The other half is getting the model to consistently produce the output your app actually needs — the right format, the right level of detail, the right tone. Poorly written prompts lead to unpredictable responses, which creates bugs that are hard to debug because the model's behavior isn't deterministic. Understanding prompt engineering gives you practical control over that behavior. For example, if you're building a chatbot that needs to always respond in JSON, or a feature that should only answer questions within a specific domain, prompt engineering techniques are what make that possible reliably. This guide gives you the vocabulary and techniques to write system prompts that actually work.

Tips for using this resource effectively

  • Read the Introduction and Basics sections first even if you've used LLMs before — the guide formalizes concepts you might have picked up informally and fills in gaps.
  • Few-shot prompting is the single most useful technique for developers building structured outputs — if you need the model to return JSON or follow a specific format, include 2-3 examples in your prompt and the results improve dramatically.
  • Use chain-of-thought prompting when your application involves any kind of reasoning or multi-step problem solving — adding "let's think step by step" or structuring the prompt to show reasoning before the answer significantly improves accuracy.
  • The guide includes runnable examples that you can test in the OpenAI Playground — try them there before adapting them to your own use case.
  • Revisit this guide whenever a feature in your app isn't behaving as expected — the issue is often the prompt structure rather than the model itself, and this resource will help you diagnose it.

Overview

LangChain is an open-source Python/JS framework for building LLM-powered applications. It provides reusable components for prompt management, model switching, memory, and data retrieval — plus LangGraph for stateful agent workflows and LangSmith for production observability.

ToolVersionWhat it doesWhen to use
LangChain (LCEL) v1.0 (Oct 2025) Components layer: prompt templates, model wrappers, output parsers. Compose with prompt | llm | parser. Simple, linear pipelines and RAG
LangGraph recommended for agents v1.1.8 Orchestration layer: stateful directed graphs, loops, branching, human-in-the-loop, node caching. Multi-step agents, complex workflows
LangSmith Hosted SaaS Logs every LLM call, input, output, and latency — with trace viewer and eval tools. Debugging and production monitoring

What's in it?

LangChain documentation is organized by abstraction layer. Here are the sections you'll use most:

  • 🔌
    Chat Models Unified interface to 50+ LLM providers (OpenAI, Anthropic, HuggingFace, Mistral, Google…). Switching providers is literally a one-line change — your app logic stays provider-agnostic.
  • 📝
    Prompt Templates + LCEL Define prompts with variable placeholders, then chain them with models and parsers using the pipe operator: chain = prompt | llm | parser. Composable, reusable, and testable.
  • 🗄️
    RAG (Retrieval-Augmented Generation) The architecture behind "chat with your documents." Built-in loaders for PDF/HTML/CSV, text splitters, embedding models, and vector store integrations (Chroma, Pinecone, FAISS).
  • 🕸️
    LangGraph Models workflows as a directed graph — each node is a function, edges are conditional. Supports node caching, deferred nodes, and pre/post hooks. All LangChain "Agents" now run on LangGraph internally.

Relevance to Your Audience

Once you're comfortable with basic API calls, LangChain is the natural next step. It turns a multi-hundred-line custom implementation into a few lines of built-in components.

  • Complex apps without complex code. A chatbot with document Q&A, multi-turn memory, and web lookup? Hundreds of lines raw — or a few built-in LangChain components.
  • Provider-agnostic architecture. Write your app logic once. Switch between OpenAI, Anthropic, or a local Hugging Face model in one line — no refactoring needed.
  • Industry-standard framework. LangChain appears constantly in AI engineering job descriptions. Knowing it demonstrates applied AI engineering sophistication beyond raw API calls.
  • Built-in RAG pipeline. The "Q&A over documents" tutorial walks you end-to-end through loading, splitting, embedding, storing, and retrieving — the most requested feature in modern AI apps.

When to use LangChain vs LangGraph

A good rule of thumb as of 2025–2026: use LangChain (LCEL) for simple, linear pipelines — a single prompt, an RAG lookup, a quick tool call. Use LangGraph for anything with state, loops, or branching — multi-step agents, workflows where you need to retry on failure, or applications where different paths run based on model output. LangGraph is production-hardened and the standard choice for agents; LangChain's older "Agent" abstraction wraps LangGraph now anyway. For observability on either, wire in LangSmith from day one.

Tips for using this resource effectively

  • Start with the Introduction and Quickstart pages to build a mental model of how LangChain's components fit together. Trying to use agents or RAG before understanding the basic LCEL chain pattern is a common mistake that leads to confusion.
  • LangChain's documentation is large and can feel overwhelming. Focus on one concept at a time — get a basic LCEL chain working, then add memory, then try RAG. Don't try to learn everything at once.
  • The How-to guides section (separate from the conceptual docs) is where you should go when you have a specific task in mind. It's organized by problem ("How to add memory to a chatbot") rather than by component, which is more useful when you're actually building something.
  • LangChain v1.0 (October 2025) established a stable API — breaking changes are now much rarer. However, always version-pin in your requirements.txt and be aware that LangGraph (currently at v1.1.8) releases independently and more frequently than LangChain itself.
  • If you're building a RAG application, the "Q&A over documents" tutorial in the How-to section is the best starting point — it walks through the full pipeline end to end with working code, covering document loading, splitting, embedding, storing, and retrieval.

Building a Chain with LCEL

Install with pip install langchain langchain-openai. The pipe operator (|) is the core LCEL syntax — it passes output from one component to the next.

from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser # 1. Define a reusable prompt template with variables prompt = ChatPromptTemplate.from_messages([ ("system", "You are a concise technical explainer."), ("human", "Explain {concept} to a {level} developer in 2 sentences.") ]) # 2. Initialize the model llm = ChatOpenAI(model="gpt-5.4-mini", temperature=0) # 3. Compose the chain with LCEL pipe syntax chain = prompt | llm | StrOutputParser() # 4. Invoke with template variables response = chain.invoke({ "concept": "vector embeddings", "level": "beginner" }) print(response) # Chains are reusable — call with different inputs without rewriting anything response2 = chain.invoke({"concept": "RAG pipelines", "level": "intermediate"}) print(response2)
